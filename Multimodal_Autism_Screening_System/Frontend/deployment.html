☁️ Deployment Guide

Deploy your ASD screening system to Heroku, AWS, or Google Cloud

📦 Deployment Options

1. Heroku (Easiest - Recommended for beginners)

Free tier available, automatic HTTPS, easy setup

Prerequisites:

  • Heroku account (free)
  • Heroku CLI installed
  • Git repository

Steps:

# Install Heroku CLI # Download from https://devcenter.heroku.com/articles/heroku-cli # Login to Heroku heroku login # Create new Heroku app heroku create multimodal-asd-screening # Add Procfile to your Backend folder echo "web: gunicorn app:app" > Procfile # Install gunicorn pip install gunicorn pip freeze > requirements.txt # Deploy git add . git commit -m "Prepare for Heroku deployment" git push heroku main # Open your app heroku open
⚠️ Note: Free tier may be slow. Consider upgrading for production use.

2. AWS (Amazon Web Services)

Scalable, reliable, industry-standard cloud platform

Using AWS Elastic Beanstalk:

# Install AWS CLI pip install awscli awsebcli # Configure AWS credentials aws configure # Initialize Elastic Beanstalk eb init -p python-3.8 multimodal-asd # Create environment and deploy eb create multimodal-env eb deploy # Open application eb open

Using AWS EC2 (Manual):

# SSH into EC2 instance ssh -i your-key.pem ec2-user@your-instance-ip # Install dependencies sudo yum update -y sudo yum install python3 python3-pip -y sudo pip3 install virtualenv # Clone your project git clone https://github.com/Shreshth-Ghub/Multimodal_Autism_Screening_System cd Multimodal_Autism_Screening_System/Backend # Setup and run python3 -m venv venv source venv/bin/activate pip install -r requirements.txt gunicorn -b 0.0.0.0:8000 app:app

3. Google Cloud Platform (GCP)

Google's cloud infrastructure with App Engine

Using App Engine:

# Install Google Cloud SDK # Download from https://cloud.google.com/sdk # Login and set project gcloud auth login gcloud config set project YOUR_PROJECT_ID # Create app.yaml file echo "runtime: python38 entrypoint: gunicorn -b :$PORT app:app" > app.yaml # Deploy gcloud app deploy # Open application gcloud app browse

🛠️ Production Configuration

Environment Variables

Create a .env file:

FLASK_ENV=production SECRET_KEY=your-secret-key-here DEBUG=False PORT=5000

CORS Configuration

Update your app.py for production:

from flask_cors import CORS app = Flask(__name__) CORS(app, origins=["https://your-frontend-domain.com"])

Gunicorn Configuration

Create gunicorn.conf.py:

workers = 4 bind = "0.0.0.0:8000" timeout = 120 keepalive = 5

📊 Monitoring & Logging

  • Use Heroku Logs: heroku logs --tail
  • AWS CloudWatch for AWS deployments
  • Google Cloud Logging for GCP
  • Consider Sentry for error tracking

🔒 Security Checklist

  • ✅ Use HTTPS (automatic on Heroku/AWS/GCP)
  • ✅ Set DEBUG=False in production
  • ✅ Use environment variables for secrets
  • ✅ Implement rate limiting
  • ✅ Add input validation
  • ✅ Keep dependencies updated
