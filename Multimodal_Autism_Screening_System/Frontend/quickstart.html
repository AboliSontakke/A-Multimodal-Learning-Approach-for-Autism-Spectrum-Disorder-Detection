🚀 Quick Start Guide

Get your ASD screening system up and running in 5 minutes

📋 Prerequisites

Before starting, make sure you have:

  • Python 3.8 or higher installed
  • Git (for cloning the repository)
  • Basic command line knowledge
  • Modern web browser (Chrome, Firefox, Edge)

⚡ Installation Steps

1Clone the Repository

Open your terminal and run:

git clone https://github.com/Shreshth-Ghub/Multimodal_Autism_Screening_System cd Multimodal_Autism_Screening_System

2Create Virtual Environment

Create and activate a Python virtual environment:

# Windows cd Backend python -m venv venv venv\Scripts\activate # Mac/Linux cd Backend python3 -m venv venv source venv/bin/activate

3Install Dependencies

Install all required Python packages:

pip install -r requirements.txt
⚠️ Note: This may take 5-10 minutes depending on your internet speed.

4Train Models (Optional)

If you want to train models from scratch:

python simple_training.py

Or use pre-trained models from saved_models/ folder.

5Start Backend Server

Launch the Flask API:

python app.py
✅ Success! Your backend is now running at http://localhost:5000

6Open Frontend

In a new terminal, navigate to frontend folder:

cd ../Frontend # Open index.html in your browser # Or use Live Server extension in VS Code

🎯 Testing the System

Upload Sample Data

  1. Go to Upload Data section
  2. Upload behavioral CSV file
  3. Upload voice recording (WAV/MP3)
  4. Upload facial image (JPG/PNG)
  5. Click Analyze Data
  6. View predictions and confidence scores

📚 API Endpoints

  • GET / - Health check
  • GET /api/models/status - Check if models are loaded
  • POST /api/predict - Make ASD prediction

🔧 Troubleshooting

Backend Not Starting?

  • Make sure virtual environment is activated
  • Check if port 5000 is already in use
  • Verify all dependencies are installed

Models Not Loading?

  • Check if saved_models/ folder exists
  • Run python simple_training.py to create models
  • Verify model files are not corrupted

Frontend Not Showing Results?

  • Check browser console for errors (F12)
  • Verify backend is running on localhost:5000
  • Check CORS settings in Flask app
